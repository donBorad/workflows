Guide to SSH into an Azure VM and Verify Network Security Group Rules

Generate the SSH keys and update the VM:

  1. Generate a new SSH key pair:
    2. 
#!/bin/bash
     ssh-keygen -t rsa -b 4096 -f ~/.ssh/new_key
  2. Update the VM's public key in the Azure portal:

    3. Go to the VM > Reset password > Reset SSH public key.
    Paste the new public key (new_key.pub).

  3. Use the new private key to connect:

    4. 
#!/bin/bash
     ssh -i ~/.ssh/new_key azureuser@48.217.88.99

Check Azure NSG (Network Security Group) Rules

Ensure that the VM's NSG allows inbound SSH traffic (port 22) from your IP address.

  1. Go to the VM in the Azure portal.
  2. Under Networking, check the inbound rules for the NSG.
  3. Ensure there's a rule allowing SSH from your IP or Any.
